O que ele faz

Duas ferramentas, ambas 100% leitura — nada é escrito no seu repositório sem você revisar.

gerar_readme

Lê target framework, pacotes NuGet, estrutura de pastas, controllers/endpoints e comentários XML, e monta um README estruturado. Onde não dá pra inferir do código (propósito de negócio), deixa um <!-- TODO --> em vez de inventar.

analisar_arquitetura

Infere o estilo arquitetural (Clean Architecture, MVC em camadas, Hexagonal) pelo grafo de ProjectReference entre os .csproj, e aponta violações de dependência entre camadas.

Por que confiar

Isso não é um SaaS fechado chamando uma IA nos bastidores. É código aberto, e o que ele faz com o seu repositório é limitado de propósito.

  • ✓
    Nenhuma IA embutida no servidorAs duas ferramentas fazem só leitura e análise determinística de código. Quem escreve a prosa (os TODOs do README) é a IA do seu próprio editor — Cursor, Claude, VS Code — não um modelo que este servidor chama ou paga.
  • ✓
    Seu código não fica armazenadoNo modo hospedado, cada chamada faz um clone raso (--depth 1) do repositório informado, analisa, e descarta o clone assim que termina.
  • ✓
    Limites de abuso aplicadosSó aceita HTTPS de github.com, gitlab.com, bitbucket.org ou dev.azure.com; timeout de 60s no clone; repositório acima de 200MB é rejeitado.
  • ✓
    Código abertoTodo o servidor está no GitHub, incluindo o próprio Dockerfile usado para hospedar isso — não tem caixa-preta.

Conectar em 30 segundos

Cole isto na configuração de MCP do seu editor — nenhuma instalação local necessária.

carregando…

Cole o JSON acima no arquivo indicado do seu cliente MCP (crie o arquivo se ele não existir).

No chat do editor, peça: "use a ferramenta gerar_readme para o repositório https://github.com/usuario/MeuProjeto".

Revise o markdown retornado, preencha os TODOs e cole no seu repositório. Nada é publicado automaticamente.
